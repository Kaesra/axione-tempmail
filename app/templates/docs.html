Authentication

Su anki access modeli

Web kullanimi

Standart register/login akisi ile cookie tabanli session acilir. Hesap panelinden API key de uretebilirsin.

API key var mi?

Evet. Kullanici hesabindan API key olusturabilir, tekrar kopyalayabilir ve istersen iptal edebilirsin. API hem `X-API-Key` hem `Authorization: Bearer` ile calisir.

Google hesap baglama var mi?

Evet. Kullanici kendi Gmail hesabini OAuth ile baglar. Sistem once `isim.soyisim@googlemail.com` gibi nokta varyasyonlu adresleri dener, yetmedigi yerde `+tag` fallback kullanir. Alias havuzu eksildiginde sistem kendini yeniden doldurur.

API

Kullanici odakli API ornekleri

1. Login ol

curl -i -X POST https://mail.axione.xyz/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{
    "username": "demo-user",
    "password": "demo-password"
  }'

Basarili login sonrasi response cookie set eder. Sonraki isteklerde ayni cookie ile devam edilir.

2. Inbox listeni cek

curl https://mail.axione.xyz/api/inboxes \
  -H 'Cookie: tempmail_session=<session-cookie>'

2b. API key ile ayni listeyi cek

curl https://mail.axione.xyz/api/inboxes \
  -H 'X-API-Key: axm_xxxxxxxxxxxxxxxxxxxxxxxxx'

3. Yeni temp inbox olustur

curl -X POST https://mail.axione.xyz/api/inboxes \
  -H 'Content-Type: application/json' \
  -H 'Cookie: tempmail_session=<session-cookie>' \
  -d '{
    "inbox_mode": "temp",
    "domain": "{{ accepted_domains[0] if accepted_domains else 'axione.xyz' }}"
  }'

4. Mesajlari listele

curl "https://mail.axione.xyz/api/inboxes/test@{{ accepted_domains[0] if accepted_domains else 'axione.xyz' }}/messages" \
  -H 'Cookie: tempmail_session=<session-cookie>'

4b. Admin olarak domain blacklist'e ekle

curl -X POST https://mail.axione.xyz/api/admin/domains/blacklist \
  -H 'Content-Type: application/json' \
  -H 'Cookie: tempmail_session=<admin-session-cookie>' \
  -d '{
    "domain": "{{ accepted_domains[0] if accepted_domains else 'example.com' }}",
    "reason": "Yeni temp olusturmayi durdur"
  }'

Blacklist'e eklenen domain UI seceneklerinden duser ve yeni temp inbox olusturma istekleri `400` ile reddedilir.

5. Tek mesaj detayini al

curl https://mail.axione.xyz/api/messages/123 \
  -H 'Cookie: tempmail_session=<session-cookie>'

Limits

Servis davranisi ve limitler

Temp inbox limiti

Standart kullanicilar gunluk belirli sayida temp inbox olusturabilir. Limit asilirsa `429` doner.

Inbox omru

Temp inbox'lar suresi dolunca temizlenebilir. Kisisel inbox'lar onaya bagli olabilir ve daha kalici davranir.

Session zorunlu

Inbox ve mesaj endpoint'lerinin tamami kullanici oturumu ister.

Health endpoint

`GET /healthz` acik saglik kontrolu icin kullanilabilir.

Domain blacklist

Admin API ile blacklist'e alinan domainler yeni inbox olusturma akisindan otomatik cikarilir.
